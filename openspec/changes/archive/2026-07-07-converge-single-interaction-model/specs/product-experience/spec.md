## ADDED Requirements

### Requirement: Product interaction contract uses a single public carrier model

Product-experience acceptance SHALL map to one canonical interaction carrier model and SHALL NOT present split L1/L2 carriers as co-equal product targets. Any pipeline-only shell path SHALL be documented as non-product verification scope.

#### Scenario: Product-experience references one canonical product carrier

- **WHEN** maintainers read product-experience acceptance mapping
- **THEN** one canonical carrier model is used for product interaction acceptance
- **AND** pipeline-only carriers are explicitly marked as non-product verification scope
