# 审查指南

审查 AetherMD 时，可按本页检查。

## 架构审查

检查提案是否：

- 保留微内核边界
- 将框架相关关注点留在 Core 之外
- 将第三方依赖 API 隔离在 Adapter 内
- 尊重当前非目标
- 明确说明质量属性或兼容性影响

## SDK 审查

检查提案是否：

- 改变插件作者行为
- 影响 Manifest 校验
- 改变生命周期顺序
- 引入新的 CapabilityId 或 PermissionId 语义
- 需要补充示例
- 后续能够被测试验证

## 工程审查

检查提案是否：

- 保持确定性的 Command 路由
- 有明确错误与恢复行为
- 可通过 telemetry 观察
- 引入并发或取消风险
- 改变 Runtime Permission 处理
- 需要新增 CI 门禁

## ADR 审查

检查 ADR 是否：

- 清楚说明背景与决策
- 记录代价，而不只记录收益
- 说明对实现和文档的后续影响
- 在替代旧决策时写明关联关系
