# 社区文档

AetherMD 正在准备成为可实现的开源项目。代码出现前，社区协作重点是让设计更清晰、更可审查、更容易落地。

## 页面

| 页面 | 作用 |
| --- | --- |
| [治理](governance.md) | 设计阶段的决策归属 |
| [审查指南](review-guide.md) | 如何审查架构、SDK 和工程文档 |

## 当前协作模型

- 改变架构约束的提案应写成 ADR。
- 影响插件作者的提案应先更新 SDK 文档。
- 工程提案应说明它满足的架构或 SDK 约束。
- 实现工作应等最小实现范围拆成里程碑后再开始。

## 待决事项

- 项目许可证
- 包管理器与 monorepo 工具
- 初始实现里程碑
- 发布与版本流程
- 进入实现阶段后的维护者角色
